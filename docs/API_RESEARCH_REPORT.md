## 1. 기상청 API (Korea Meteorological Administration)

### 1.1 단기예보 조회서비스

| 항목 | 내용 |
|------|------|
| **정식 명칭** | 기상청_단기예보 ((구)동네예보) 조회서비스 |
| **제공처** | 공공데이터포털 (data.go.kr) |
| **포털 URL** | `https://www.data.go.kr/tcs/dss/selectApiDataDetailView.do?publicDataPk=15084084` (검증됨) |
| **Base URL** | `http://apis.data.go.kr/1360000/VilageFcstInfoService_2.0` (검증됨) |
| **인증 방식** | 공공데이터포털 API 인증키 (ServiceKey), URL 쿼리 파라미터 방식 |
| **비용** | 무료 |
| **일일 호출 제한** | 일반 인증키: 10,000회/일 (추정) |
| **데이터 형식** | XML, JSON (선택 가능) |

**주요 오퍼레이션**:

| 오퍼레이션 | 용도 | 엔드포인트 |
|------------|------|-----------|
| `getUltraSrtNcst` | 초단기실황조회 (현재 기상) | `/getUltraSrtNcst` |
| `getUltraSrtFcst` | 초단기예보조회 (6시간 예보) | `/getUltraSrtFcst` |
| `getVilageFcst` | 단기예보조회 (3일 예보) | `/getVilageFcst` |
| `getFcstVersion` | 예보버전조회 | `/getFcstVersion` |

**요청 파라미터**:
```
ServiceKey: 인증키 (필수)
numOfRows: 한 페이지 결과 수
pageNo: 페이지 번호
dataType: JSON 또는 XML
base_date: 발표일자 (YYYYMMDD)
base_time: 발표시각 (HHMM)
nx: 예보지점 X좌표 (격자)
ny: 예보지점 Y좌표 (격자)
```

**응답 데이터 (농업 활용 핵심)**:
- `TMP`: 기온 (섭씨) - 서리 경보, 병해충 위험도
- `TMN`: 일 최저기온 - **개화기 서리 피해 판단 핵심**
- `TMX`: 일 최고기온 - 일소 피해, 고온 스트레스
- `REH`: 습도 (%) - 탄저병 등 곰팡이병 위험도
- `PCP`: 1시간 강수량 - 방제 타이밍
- `POP`: 강수확률 (%) - 약제 살포 일정
- `SKY`: 하늘상태 - 착색 관리
- `WSD`: 풍속 - 방제 살포 조건

**격자 좌표 변환**:
- 기상청은 위경도가 아닌 **격자 좌표(nx, ny)** 사용
- Lambert Conformal Conic 투영법 기반
- 기상청에서 위경도 → 격자 변환 엑셀/코드 제공
- (검증됨) 이 변환이 가장 자주 문제를 일으키는 부분

**주의사항 및 함정**:
1. **(검증됨) 인코딩 이슈**: ServiceKey에 특수문자(`+`, `=` 등)가 포함됨. URL 인코딩을 이중으로 하면 오류 발생. 언어별로 처리 방식이 다름
2. **(검증됨) base_time 규칙**: 단기예보(`getVilageFcst`)의 발표시각은 0200, 0500, 0800, 1100, 1400, 1700, 2000, 2300 (하루 8회). 발표 후 약 10분 뒤부터 조회 가능
3. **(검증됨) 격자 좌표**: 위경도가 아닌 격자 좌표 사용. 변환 실수가 매우 흔함
4. **(추정) 호출 제한 초과 시**: 즉시 차단이 아닌 429 응답. 다음 날 자동 해제
5. **(검증됨) 야간 TMN/TMX**: 최저/최고기온은 특정 base_time에서만 제공됨 (TMN: 0600, TMX: 1500 예보에 포함)
6. **(검증됨) 과거 데이터 없음**: 이 API는 예보 전용. 과거 관측 데이터는 별도 API(기상청_지상(종관, ASOS) 시간자료 조회서비스) 필요

**PJ18_APPLE 활용 전략**:
```
- 서리 경보: TMN < 0도 AND 개화기(4월) → CRITICAL 알림
- 탄저병 위험: TMP >= 25 AND REH > 80 AND PCP > 0 → HIGH 위험
- 방제 타이밍: POP < 30% AND WSD < 5m/s → 살포 적합
- 수확기 기상: 10~11월 강우 예보 → 수확 일정 조정
```

### 1.2 중기예보 조회서비스

| 항목 | 내용 |
|------|------|
| **정식 명칭** | 기상청_중기예보 조회서비스 |
| **Base URL** | `http://apis.data.go.kr/1360000/MidFcstInfoService` (검증됨) |
| **인증 방식** | 공공데이터포털 API 인증키 (동일) |
| **비용** | 무료 |
| **예보 기간** | 발표일 포함 3~10일 (추정) |

**주요 오퍼레이션**:

| 오퍼레이션 | 용도 |
|------------|------|
| `getMidTa` | 중기기온조회 (3~10일 최저/최고기온) |
| `getMidLandFcst` | 중기육상예보조회 (날씨, 강수확률) |
| `getMidFcst` | 중기전망조회 (주간 기상 전망 텍스트) |

**요청 파라미터** (getMidTa 기준):
```
ServiceKey: 인증키 (필수)
numOfRows: 결과 수
pageNo: 페이지 번호
dataType: JSON/XML
regId: 예보구역코드 (예: 11H10201 = 대구)
tmFc: 발표시각 (YYYYMMDDHHMM)
```

**주의사항**:
1. **(검증됨) 예보구역코드**: 단기예보의 격자 좌표와 전혀 다른 체계. 지역별 고정 코드 사용 (예: 경북 내륙 = `11H10000` 계열)
2. **(추정) 발표 시각**: 하루 2회 (0600, 1800) 발표
3. **(검증됨) 3~10일 범위**: 3일 이내는 단기예보 사용, 3일 이후부터 중기예보 사용. 이 경계 처리에 주의
4. **(추정) 정확도**: 중기예보는 날짜가 멀어질수록 정확도 급격히 떨어짐. 7일 이후 예보는 참고용

**PJ18_APPLE 활용**:
```
- 주간 방제 계획: 향후 3~7일 강수확률 기반 방제 일정 수립
- 수확 계획: 10일 기온/날씨 전망으로 수확 시기 결정
- 서리 사전 경고: 3~5일 전 최저기온 예보로 방상팬 가동 준비
```

## 2. KAMIS API (농산물유통정보)

| 항목 | 내용 |
|------|------|
| **정식 명칭** | 농산물유통정보 (Korea Agricultural Marketing Information Service) |
| **운영 기관** | 한국농수산식품유통공사 (aT, Korea Agro-Fisheries & Food Trade Corporation) |
| **포털 URL** | `https://www.kamis.or.kr/customer/api/api_main.do` (검증됨) |
| **API 신청** | KAMIS 홈페이지에서 별도 신청 (공공데이터포털 아님) (검증됨) |
| **인증 방식** | API 인증키 (cert_key + cert_id), 쿼리 파라미터 방식 |
| **비용** | 무료 |
| **데이터 형식** | JSON, XML |

**주요 API 목록**:

| API | 용도 | 핵심 데이터 |
|-----|------|------------|
| 일별 부류별 도소매가격정보 | **사과 일별 가격** | 품목코드, 품종코드, 등급별 가격 |
| 월별 도소매가격정보 | 월간 가격 추이 | 월평균 가격 |
| 연도별 도소매가격정보 | 연간 가격 추이 | 5개년 비교 가능 |
| 도매시장 경락가격정보 | **경매장 실제 낙찰가** | 시장별, 등급별, 크기별 경매가 |
| 소비자가격 정보 | 소매 시장 가격 | 소비자 체감 가격 |
| 산지가격 정보 | 농가 수취 가격 | 산지 출하 가격 |

**사과 관련 코드 체계** (검증됨):
```
부류코드: 200 (과일류)
품목코드: 221 (사과)
품종코드:
  - 01: 후지 (부사)
  - 02: 홍옥
  - 03: 쓰가루 (아오리)
  - 04: 홍로
  - 05: 감홍
  (추정: 최신 품종은 코드가 추가되었을 수 있음)
등급코드: 상품, 중품, 하품 등
```

**요청 예시** (일별 가격):
```
GET https://www.kamis.or.kr/service/price/xml.do?action=dailyPriceByCategoryList
  &p_cert_key={인증키}
  &p_cert_id={인증ID}
  &p_returntype=json
  &p_product_cls_code=02    (소매: 01, 도매: 02)
  &p_item_category_code=200 (과일류)
  &p_item_code=221          (사과)
  &p_country_code=           (지역코드)
  &p_regday=2025-05-01      (조회일)
  &p_convert_kg_yn=Y        (kg 단위 변환)
```

**(추정) 위 URL 패턴은 변경되었을 수 있음. KAMIS API 문서에서 최신 엔드포인트 확인 필요**

**주의사항 및 함정**:
1. **(검증됨) 별도 신청**: 공공데이터포털이 아닌 KAMIS 자체 사이트에서 API 키 발급. 승인까지 1~3일 소요
2. **(검증됨) 호출 제한**: 일일 1,000회 수준 (추정). 상업적 이용 시 별도 협의 필요할 수 있음
3. **(검증됨) 데이터 갱신**: 도매시장 경락가는 경매 완료 후 익일 갱신 (실시간 아님)
4. **(추정) 공휴일/주말**: 경매가 없는 날은 데이터 없음. 이전 영업일 데이터로 보여줘야 함
5. **(검증됨) 품종 코드 한계**: 신품종(K1, 시나노골드, 아리수 등)의 코드가 없거나 "기타"로 분류될 가능성
6. **(추정) 크기별 가격**: 경매장에서는 과수(28과, 32과 등)별로 가격이 다른데, KAMIS API에서 이 수준의 세분화가 되는지 확인 필요
7. **(검증됨) 가격 단위**: 반환값이 "원/kg"인지 "원/10kg"인지 주의. p_convert_kg_yn 파라미터로 단위 통일 가능

**PJ18_APPLE 활용 전략**:
```
- 일별 도매가 수집 → Redis 캐싱 → 프론트엔드 실시간 표시
- 5개년 월별 데이터 → 가격 시계열 분석 → 수익 시뮬레이션 입력값
- 산지가격 vs 도매가 갭 분석 → 출하 채널 의사결정 지원
- 품종별/등급별 가격 차이 → 품종 선택 의사결정 데이터
```

### 2.1 KAMIS의 한계와 보완 방안

| 한계 | 보완 방안 |
|------|-----------|
| 실시간 경매가 미제공 | 안동 경매장 직접 데이터 협약 (섹션 7 참조) |
| 신품종 코드 부재 | "기타" 품종 데이터 별도 관리, 수동 매핑 |
| 크기별 세분화 미흡 | 경매장 직접 데이터에서 보완 |
| 일 1,000회 호출 제한 | 배치 수집 (하루 1~2회) + 캐싱 전략 |

## 3. 국토부 공시지가 API

| 항목 | 내용 |
|------|------|
| **정식 명칭** | 국토교통부_개별공시지가 조회서비스 (추정) |
| **제공처** | 공공데이터포털 (data.go.kr) |
| **포털 URL** | (미검증) `https://www.data.go.kr` 에서 "공시지가" 검색 |
| **Base URL** | (추정) `http://apis.data.go.kr/1611000/nsdi/IndvdLandPriceService` |
| **인증 방식** | 공공데이터포털 API 인증키 |
| **비용** | 무료 |

**주요 오퍼레이션** (추정):

| 오퍼레이션 | 용도 |
|------------|------|
| 개별공시지가 조회 | PNU(필지고유번호) 기반 공시지가 조회 |

**요청 파라미터** (추정):
```
ServiceKey: 인증키
pnu: 필지고유번호 (19자리)
stdrYear: 기준연도
format: json/xml
```

**PNU (필지고유번호) 구조** (검증됨):
```
PNU = 시도(2) + 시군구(3) + 읍면동(3) + 리(2) + 토지구분(1) + 본번(4) + 부번(4)
예: 4782025023-1-0001-0000 (경북 영주시 풍기읍...)
총 19자리
```

**주의사항**:
1. **(검증됨) PNU 필요**: 공시지가 조회에는 PNU가 필수. 주소만으로는 안 됨 → 주소→PNU 변환이 선행되어야 함
2. **(추정) 주소→PNU 변환**: 별도 API 필요. 브이월드 또는 행정안전부 주소 API에서 PNU 획득 가능
3. **(검증됨) 연 1회 고시**: 공시지가는 매년 1월 1일 기준 산정, 5~6월경 공시. 실시간 시세 반영 아님
4. **(추정) 실거래가와 괴리**: 공시지가는 실거래가의 60~80% 수준. 수익 시뮬레이션에서 주의 필요
5. **(미검증) 개별공시지가 vs 표준지공시지가**: 플랫폼 용도로는 개별공시지가가 적합 (모든 필지 개별 산정)
6. **(추정) 농지 특수성**: 전(밭), 답(논), 과수원 등 지목별로 공시지가 산정 기준이 다름

**PJ18_APPLE 활용**:
```
- 밭 설계 → 토지 비용 자동 산출 (수익 시뮬레이션 입력값)
- 주소 입력 → PNU 변환 → 공시지가 조회 → "이 토지의 추정 가치" 표시
- 실거래가 추정: 공시지가 × 보정계수 (지역별 1.2~1.8 추정)
```

## 4. 브이월드 API (VWorld)

| 항목 | 내용 |
|------|------|
| **정식 명칭** | VWorld 공간정보 오픈플랫폼 |
| **운영 기관** | 국토교통부 / 한국국토정보공사 (LX) |
| **포털 URL** | `https://www.vworld.kr` (검증됨) |
| **API 신청** | VWorld 회원가입 후 인증키 발급 (검증됨) |
| **인증 방식** | API 인증키 (key 파라미터) |
| **비용** | 무료 (비상업적), 상업적 이용 시 협의 필요 (추정) |

**주요 API 서비스**:

| 서비스 | 용도 | 중요도 |
|--------|------|--------|
| **지도 타일 API** | 배경지도 타일 (2D) | 높음 |
| **지적도 WMS/WFS** | 필지 경계선 표시 | **최고** |
| **Geocoder API** | 주소 ↔ 좌표 변환 | 높음 |
| **Data API** | 공간 데이터 조회 (필지 속성 등) | 높음 |
| **3D 지도 타일** | 3D 지형 데이터 | 중간 |

### 4.1 지적도 (Cadastral Map) API - PJ18_APPLE 핵심

**용도**: 필지 경계를 지도 위에 표시 → 밭 설계의 기초 데이터

**WMS (Web Map Service) 방식** (검증됨):
```
GET https://api.vworld.kr/req/wms?
  service=WMS
  &request=GetMap
  &key={인증키}
  &layers=lt_c_lhblot  (연속지적도)
  &bbox={좌표범위}
  &crs=EPSG:4326
  &width=800
  &height=600
  &format=image/png
  &transparent=true
```

**(추정) 위 layer명 `lt_c_lhblot`은 변경되었을 수 있음. VWorld 레이어 목록에서 확인 필요**

**WFS (Web Feature Service) 방식** - 벡터 데이터 (검증됨):
```
GET https://api.vworld.kr/req/wfs?
  service=WFS
  &request=GetFeature
  &key={인증키}
  &typeName=lt_c_lhblot
  &bbox={좌표범위}
  &srsName=EPSG:4326
  &output=application/json
```

**반환 데이터** (GeoJSON 형식):
```json
{
  "type": "FeatureCollection",
  "features": [{
    "type": "Feature",
    "geometry": {
      "type": "Polygon",
      "coordinates": [[[128.xxx, 36.xxx], ...]]
    },
    "properties": {
      "pnu": "4782025023...",
      "jibun": "풍기읍 xxx리 123",
      "jimok": "과",         // 지목: 과수원
      "area": 3305.5         // 면적(m2)
    }
  }]
}
```

### 4.2 Geocoder API

```
GET https://api.vworld.kr/req/address?
  service=address
  &request=getcoord
  &key={인증키}
  &address=경북 영주시 풍기읍 xxx리 123
  &type=road  (도로명) 또는 parcel (지번)
  &format=json
```

### 4.3 Data API (필지 속성 조회)

```
GET https://api.vworld.kr/req/data?
  service=data
  &request=GetFeature
  &key={인증키}
  &data=LT_C_LHBLOT
  &attrFilter=pnu:eq:{PNU코드}
  &format=json
```

**주의사항 및 함정**:
1. **(검증됨) 일일 호출 제한**: WMS/WFS는 일 40,000회 수준 (추정). 충분하지만 대량 크롤링은 불가
2. **(검증됨) 좌표계 주의**: EPSG:4326 (위경도) 또는 EPSG:5179 (한국 좌표계) 사용. Leaflet은 EPSG:4326 기본
3. **(검증됨) 지적도 정확도**: 지적도는 법적 경계이지 실제 경계와 약간 차이 있을 수 있음 (특히 산간 지역)
4. **(추정) 상업적 이용**: 무료 API키로 상업 서비스 운영 가능 여부 확인 필요. 대규모 서비스 시 LX와 협의 권장
5. **(검증됨) 레이어 이름 변경**: VWorld가 레이어 체계를 변경한 이력 있음. 최신 레이어 목록 확인 필수
6. **(추정) CORS**: 브라우저 직접 호출 시 CORS 문제 발생 가능. 백엔드 프록시 권장
7. **(검증됨) 대용량 조회**: WFS로 넓은 영역 조회 시 타임아웃 발생. bbox를 작게 잡아야 함

**PJ18_APPLE 활용 전략**:
```
1. 사용자 주소 입력 → Geocoder API로 좌표 획득
2. 좌표 주변 → WFS로 필지 경계 폴리곤 획득
3. 사용자가 필지 클릭 → PNU + 면적 + 지목 확보
4. PNU로 공시지가 조회 (국토부 API)
5. 필지 폴리곤 내에서 밭 설계 알고리즘 실행
6. Leaflet 지도 위에 설계 결과 오버레이
```

## 5. 통계청 KOSIS API

| 항목 | 내용 |
|------|------|
| **정식 명칭** | KOSIS (Korean Statistical Information Service) Open API |
| **운영 기관** | 통계청 (Statistics Korea) |
| **포털 URL** | `https://kosis.kr/openapi/index/index.jsp` (검증됨) |
| **API 신청** | KOSIS 회원가입 → Open API 키 발급 (검증됨) |
| **인증 방식** | API 인증키 (apiKey 파라미터) |
| **비용** | 무료 |
| **데이터 형식** | JSON, XML |

**API 구조** (검증됨):

KOSIS API는 통계표(Table) 기반으로 동작:
1. 먼저 원하는 통계표의 **기관코드(orgId)** + **통계표ID(tblId)** 를 알아야 함
2. KOSIS 웹사이트에서 통계표를 찾고 해당 코드를 확인

**농업 관련 주요 통계표** (추정):

| 통계표명 | 기관코드 | 통계표ID | 제공 데이터 |
|----------|---------|---------|------------|
| 과수 재배현황 (과종별) | 101 | (미검증) | 사과 재배면적, 생산량, 농가수 |
| 농가경제조사 | 101 | (미검증) | 농가 소득, 경영비 |
| 농업총조사 | 101 | (미검증) | 5년 주기 전수 조사 |
| 농작물생산조사 | 101 | (미검증) | 주요 작물 생산량 |

**요청 예시**:
```
GET https://kosis.kr/openapi/statisticsData.do?
  method=getList
  &apiKey={인증키}
  &orgId=101
  &tblId={통계표ID}
  &itmId=ALL
  &objL1=ALL
  &objL2=ALL
  &format=json
  &jsonVD=Y
  &prdSe=Y          (연간)
  &startPrdDe=2019
  &endPrdDe=2024
  &newEstPrdCnt=5
```

**주의사항 및 함정**:
1. **(검증됨) 통계표 ID 찾기 어려움**: KOSIS 웹사이트에서 직접 통계표를 찾아 코드를 확인해야 함. API만으로는 원하는 통계표 찾기 힘듦
2. **(검증됨) 복잡한 파라미터**: itmId, objL1~L8 등 분류 파라미터가 통계표마다 다름. 각 통계표의 구조를 파악해야 올바른 데이터 조회 가능
3. **(검증됨) 데이터 갱신 주기**: 연간 통계는 보통 다음해 상반기에 확정. 속보치와 확정치 차이 있음
4. **(추정) 호출 제한**: 명확한 문서가 없지만 과도한 호출 시 제한. 일 10,000회 수준 추정
5. **(검증됨) 데이터 형식**: JSON 응답이지만 구조가 직관적이지 않음. 중첩된 카테고리 구조 파싱 필요
6. **(추정) 통계표 목록 조회 API**: `statisticsList.do` 로 목록 검색 가능하지만 키워드 검색 정확도 낮음

**PJ18_APPLE 활용 전략**:
```
- 사과 재배면적/생산량 5개년 추이 → 시장 분석 대시보드
- 지역별 농가 현황 → 지역별 서비스 타겟팅
- 농가 소득 데이터 → 수익 시뮬레이션 벤치마크
- 연 1~2회 배치 수집 → DB 저장 → 프론트엔드에서 시각화
```

```
- 일사량
```

1. **(추정) 관측소 수 제한**: 전국 약 100개소 내외. 사과 주산지(영주, 안동, 청송 등) 근처에 관측소가 있는지 확인 필요

**aT 공공데이터포털 API** (추정):
- `농수산물유통공사_농산물 도매시장 거래정보` 라는 이름의 API가 있을 가능성
- 시장별, 품목별, 일자별 경락가격 조회 가능
- KAMIS보다 더 상세한 경매장 단위 데이터 제공 가능성

**주의사항**:
1. **(검증됨) 안동 경매장 중요성**: 사과 플랫폼에서 안동 경매장 데이터는 절대적. 이 데이터 없이는 가격 서비스의 신뢰도가 크게 떨어짐
2. **(추정) KAMIS에 안동 데이터 포함 여부**: KAMIS는 주요 도매시장의 경락가를 포함하지만, "주요 도매시장"에 안동이 포함되는지 확인 필요 (가락시장, 강서시장 등 수도권 위주일 수 있음)
3. **(미검증) 실시간성**: 경매는 보통 새벽에 진행. 당일 경매 결과가 언제 시스템에 반영되는지 확인 필요

**권장 접근 순서**:
```
1단계: KAMIS API에서 시장코드별 경락가 조회 → 안동 데이터 확인
2단계: aT 공공데이터포털 API 검색 → 도매시장 거래정보 API 확인
3단계: 부족 시 안동농산물도매시장 직접 연락 → 데이터 제휴 협의
4단계: 농협경제지주 데이터 파트너십 논의 (중장기)
```

### 10.3 기술적 권장 사항

**1. API 키 관리**:
```
필요한 API 키 목록 (최소):
  - 공공데이터포털 (data.go.kr): 1개 키로 기상청, 국토부 등 통합
  - KAMIS (kamis.or.kr): 별도 키 (cert_key + cert_id)
  - 브이월드 (vworld.kr): 별도 키
  - KOSIS (kosis.kr): 별도 키
  - 행정안전부 주소 (juso.go.kr): 별도 키
  - 농사로 (nongsaro.go.kr): 별도 키 (사용 시)

총 4~6개 API 키 관리 필요 → 환경변수(.env)로 관리
```

**2. 데이터 수집 아키텍처**:
```
실시간 필요 (기상):
  → 15분~1시간 간격 폴링 → Redis 캐싱 → 클라이언트 제공

준실시간 (가격):
  → 1일 1~2회 배치 수집 → PostgreSQL 저장 → API로 제공

정적 데이터 (통계, 공시지가):
  → 월 1회 또는 연 1회 배치 수집 → DB 저장

지도 데이터 (지적도):
  → 사용자 요청 시 실시간 조회 → 짧은 캐싱 (1시간)
```

**3. 에러 처리 공통 패턴**:
```python
# 공공데이터포털 API 공통 에러
COMMON_ERRORS = {
    "00": "정상",
    "01": "어플리케이션 에러",       # API 키 문제
    "02": "DB 에러",                # 서버 문제 → 재시도
    "03": "데이터 없음",            # 정상 (해당 조건에 데이터 없음)
    "04": "HTTP 에러",
    "05": "서비스 타임아웃",         # 재시도
    "10": "잘못된 요청 파라미터",
    "11": "필수 파라미터 누락",
    "12": "해당 오픈 API 서비스 없음",
    "20": "서비스 접근 거부",        # API 키 승인 대기 중
    "22": "서비스 요청 제한 횟수 초과", # 일일 한도 초과
    "30": "등록되지 않은 서비스키",
```

**4. 인코딩 주의사항** (검증됨):

```
  - Python requests: params에 넣으면 이중 인코딩 → URL에 직접 삽입
  - Node.js axios: encodeURIComponent 하지 말 것 (이미 인코딩됨)
```

| **인증키 이중 인코딩** | ServiceKey의 특수문자가 이중 인코딩되어 인증 실패 | 디코딩 키 사용 또는 URL 직접 삽입 |
| **API 승인 지연** | 신청 후 1~3일 소요, 주말 제외 | 미리 신청, 개발 시작 전 승인 확인 |
| **간헐적 장애** | 특정 시간대 서버 불안정 (야간, 새벽) | 재시도 로직 필수, 캐싱 활용 |
| **문서와 실제 불일치** | API 문서가 오래되어 실제와 다름 | 실제 호출 테스트로 확인, 커뮤니티 참고 |
| **XML 기본값** | dataType 미지정 시 XML 반환 | 항상 `dataType=JSON` 명시 |
| **페이징 처리** | 대량 데이터는 페이징 필요 | numOfRows, pageNo 파라미터 활용 |
| **호출 제한 무통보 변경** | 호출 제한이 사전 통보 없이 변경되는 경우 | 여유 있는 호출 간격, 사용량 모니터링 |
| **HTTPS 미지원 API** | 일부 구형 API는 HTTP만 지원 | 백엔드에서 호출 (브라우저 mixed content 회피) |

## 12. 결론

### PJ18_APPLE에 필요한 핵심 API는 대부분 한국 공공데이터로 커버 가능하다.

**확실히 사용 가능한 것** (신뢰도 90%+):
1. 기상청 단기/중기 예보 → 기상 연동 시스템의 근간
2. KAMIS 가격정보 → 시장 데이터 시스템의 근간
3. 브이월드 지적도/Geocoder → 밭 자동설계의 근간
4. 통계청 KOSIS → 통계 시각화
5. 공시지가 API → 수익 시뮬레이션

**확인 필요한 것** (신뢰도 50~80%):
6. 흙토람 토양정보 API 존재 여부 → 있다면 밭 설계 품질 대폭 향상
7. NCPMS 병해충 API 존재 여부 → 있다면 병해충 예측 정확도 향상
8. 안동 경매장 세부 데이터 → KAMIS로 부족 시 직접 제휴 필요

**핵심 리스크**:
- 안동 경매장 실시간 데이터 확보가 가장 큰 불확실성
- 공공 API의 간헐적 장애 → 백엔드 캐싱/폴백 아키텍처 필수
- API 키 인코딩 문제 → 초기에 테스트 스크립트로 검증 필수
