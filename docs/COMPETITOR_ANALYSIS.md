## 1. 국내 농업 플랫폼

### 1.1 팜모닝 (FarmMorning) - 그린랩스

| 항목 | 내용 |
|------|------|
| **운영사** | 그린랩스 (GreenLabs) |
| **URL** | https://www.farmmorning.com / 앱스토어 |
| **출시** | 2017년 |
| **사용자 수** | 누적 가입자 약 100만+ (2024년 기준, 국내 농업 앱 1위) |
| **투자 규모** | 시리즈 C 이후 총 투자 약 2,000억원+ (2022년 기준) |
| **비즈니스 모델** | 앱 무료, 농자재 이커머스(팜모닝몰), 광고, B2B 데이터 |

#### 핵심 기능

1. **농업 커뮤니티 (핵심)**
   - 작물별 커뮤니티 게시판 (사과, 배, 벼, 고추 등)
   - 농업인 간 사진/동영상 기반 Q&A
   - "오늘의 농장" 형태의 일지/SNS
   - 댓글/좋아요 등 소셜 기능

2. **기상 정보**
   - 필지 기반 날씨 예보
   - 기상 알림 (서리, 폭우, 폭염)
   - 시간대별 기온/강수 예보

3. **농약 검색**
   - 작물별 등록 농약 검색
   - 안전사용기준 조회
   - PLS (허용물질목록관리제도) 대응

4. **농산물 시세**
   - KAMIS 연동 도/소매 가격
   - 품목별 시세 추이

5. **팜모닝몰**
   - 농자재 (비료, 농약, 자재) 직접 판매
   - 가격 비교, 배송

6. **영농일지**
   - 작업 기록, 사진 첨부
   - 방제 이력 관리

#### 기술 스택 (추정)
- 모바일: React Native 또는 Flutter (크로스플랫폼)
- 백엔드: 마이크로서비스 아키텍처 (AWS 기반)
- 데이터: 기상청 API, KAMIS API 등 공공데이터 연동

#### 강점
- **압도적 사용자 수**: 농업인 앱 중 1위, 네트워크 효과
- **커뮤니티 활성화**: 실제 농업인 참여 활발, 체류 시간 높음
- **농약 검색**: PLS 대응으로 실질적 가치
- **대규모 투자**: 자금력으로 빠른 기능 확장 가능

#### 약점 (PJ18_APPLE 기회)
- **사과 전문성 부족**: 범용 플랫폼으로 모든 작물 다룸, 사과 특화 깊이 없음
- **밭 설계 기능 없음**: 과수원 레이아웃/설계 도구 전무
- **수익 시뮬레이션 없음**: 투자 대비 수익 분석 기능 없음
- **경매가 분석 부족**: 단순 시세 표시만, 분석/예측 없음
- **병해충 예측 약함**: 기상 연동 예측 모델보다 단순 정보 제공 수준
- **3D 시각화 없음**: 텍스트/이미지 기반, 시각적 설계 도구 없음

#### 교훈 (PJ18_APPLE 적용)
- 커뮤니티의 힘: 사용자 유입/유지에 커뮤니티가 핵심
- 농약 검색은 필수 기능 (PLS 대응)
- 모바일 우선 설계 검증됨

#### 핵심 기능 (사과 관련)

1. **작물별 재배 기술**
   - 사과 재배 전반 매뉴얼 (PDF/HTML)
   - 품종별 특성 설명
   - 월별 관리 요령

2. **병해충 정보**
   - 사과 주요 병해충 목록
   - 사진 기반 병해충 도감
   - 방제 방법 텍스트 설명

3. **농업기술 영상**
   - 재배 기술 동영상 (유튜브 연계)
   - 전문가 강의

4. **영농기술 상담**
   - 온라인 Q&A 게시판
   - 전문가 답변

5. **농업날씨 365**
   - 농업기상 정보
   - 지역별 농업 기상 관측 데이터

6. **농업경영 정보**
   - 경영비 분석 자료
   - 품목별 경영 데이터

#### 강점
- **공신력**: 정부 기관 자료로 신뢰도 최고
- **깊이 있는 기술 정보**: 연구 기반 재배 기술
- **완전 무료**: 접근 장벽 없음
- **사진 도감**: 병해충 사진 DB 풍부

#### 약점 (PJ18_APPLE 기회)
- **UI/UX 극도로 불편**: 2000년대 초반 수준의 인터페이스, 정보 찾기 어려움
- **모바일 대응 미흡**: 반응형 웹 부재 또는 불완전
- **실시간성 없음**: 정적 콘텐츠, 기상 연동 실시간 알림 없음
- **개인화 없음**: 내 밭/내 품종 기반 맞춤 정보 불가
- **분석 도구 없음**: 데이터 나열만, 분석/시뮬레이션 전무
- **업데이트 느림**: 정부 기관 특성상 콘텐츠 갱신 주기 길음
- **밭 설계 기능 없음**: 토지 활용 계획 도구 전무

#### 전략적 활용
- 농사로의 기술 데이터를 PJ18_APPLE의 품종 DB 구축 시 참고 자료로 활용
- 농사로가 제공하지 못하는 "UX + 실시간 + 분석"을 PJ18_APPLE이 제공

## 2. 국내 농업 데이터 서비스

### 2.1 KAMIS (농산물유통정보)

| 항목 | 내용 |
|------|------|
| **운영** | 한국농수산식품유통공사 (aT) |
| **URL** | https://www.kamis.or.kr |
| **비용** | 무료 |
| **API** | 공공데이터포털에서 API 제공 |
| **주 용도** | 농산물 도매/소매 가격 정보 |

#### 제공 데이터 (사과 관련)

1. **도매 가격**
   - 전국 주요 도매시장 (가락, 대구, 부산 등) 거래가격
   - 품종별 (후지/홍로 등), 등급별 (특/상/보통), 크기별
   - 일일/주간/월간/연간 추이

2. **소매 가격**
   - 전국 주요 소비지역 소매 가격
   - 대형마트/전통시장 구분

3. **반입 물량**
   - 도매시장 반입 물량 데이터
   - 물량 대비 가격 추이

4. **생산자 가격**
   - 산지 가격 정보 (제한적)

#### 웹 서비스 사용성 평가

| 항목 | 점수 (5점) | 설명 |
|------|------------|------|
| 데이터 정확성 | 4.5 | 공식 경매 데이터, 신뢰도 높음 |
| 업데이트 빈도 | 4.0 | 일일 업데이트, 실시간은 아님 |
| UI/UX | 2.0 | 구식 인터페이스, 직관성 낮음 |
| 모바일 대응 | 2.0 | 반응형 웹 미흡 |
| 검색/필터 | 2.5 | 기능은 있으나 사용성 불편 |
| 시각화 | 2.0 | 기본적인 표/차트만, 분석 도구 부재 |
| API 품질 | 3.0 | 공공데이터포털 API 제공, 문서화 보통 |
| 개인화 | 1.0 | 즐겨찾기/알림 등 개인화 기능 없음 |

#### 약점 (PJ18_APPLE 기회)
- **분석 기능 부재**: 가격 데이터만 나열, 트렌드 분석/예측 없음
- **알림 없음**: 가격 변동 알림, 출하 적기 알림 등 없음
- **품종 세분화 부족**: 사과 전체 묶음, 세부 품종별 구분 제한적
- **기상 연동 없음**: 가격과 기상의 상관관계 분석 없음
- **시각화 미흡**: 복합 차트, 비교 분석 등 고급 시각화 없음

#### PJ18_APPLE 전략
- KAMIS API를 데이터 소스로 활용하되, **분석/예측/시각화 레이어**를 추가
- "KAMIS 데이터 + 기상 데이터 + AI 분석 = 출하 시기 추천" 형태의 부가가치 창출
- KAMIS가 제공하지 못하는 개인화 알림, 품종별 세분 분석 제공

- KAMIS보다 더 **원시 데이터**에 가까움 (경매 건별 데이터)

- UI가 KAMIS보다 더 불편하고 투박함
- 데이터 접근성이 낮음 (웹 크롤링 수준)

#### PJ18_APPLE 기회

- 안동 경매장 특화 분석 (사과 물량 60%+ 체결)
- 경매 건별 분석 → 크기별/등급별 가격 트렌드 도출

### 4.1 과수원/농장 자동 레이아웃 설계

**현황**: 과수원 레이아웃을 자동으로 설계해주는 전용 소프트웨어는 **상용 시장에 사실상 존재하지 않음**.

#### 유사 도구들

| 도구 | 분야 | 관련성 | 설명 |
|------|------|--------|------|
| **QGIS** | 오픈소스 GIS | 중간 | 범용 GIS, 농업 플러그인 존재하나 자동 설계 아님 |
| **ArcGIS Pro** | 상용 GIS | 중간 | Esri 사, 정밀 농업 분석 가능하나 자동 설계 아님, 매우 고가 |
| **John Deere Operations Center** | 정밀 농업 | 낮음 | 농기계 연동 필지 관리, 자동 설계 아님 |
| **Farmable** | 노르웨이 | 중간 | 필지 관리 앱, 과수원 지원하나 자동 설계 아님 |
| **SketchUp / CAD** | 범용 설계 | 낮음 | 수동 설계 가능하나 농업 특화 아님 |

#### 학술/연구 영역

- **Parking Lot Layout Optimization** 논문들: 주차장 자동 설계 알고리즘이 과수원 나무 배치와 유사한 수학적 구조
- **Vineyard Design Software**: 포도원 설계 연구 (열 방향, 간격 최적화) 논문 존재
- **Orchard System Design**: 워싱턴 주립대, 코넬대 등에서 과수원 설계 가이드라인 제공하나 소프트웨어는 아님
- **Precision Agriculture Research**: 학술적으로 "optimal tree placement" 연구 존재하나 상용화 사례 없음

#### 결론
- **과수원 자동 설계 도구는 글로벌 시장에서도 사실상 블루오션**
- 학술 연구는 있으나 사용자 친화적 소프트웨어로 구현된 사례 없음
- PJ18_APPLE의 밭 자동설계 기능은 **세계적으로도 독보적인 포지셔닝** 가능

## 7. 결론 및 전략적 시사점

### 7.1 핵심 발견

1. **글로벌 블루오션 확인**: 과수원 자동 설계 + 수익 시뮬레이션 도구는 국내는 물론 해외에서도 상용 서비스가 없음. PJ18_APPLE의 킬러 피처가 글로벌 차원에서도 독보적임.

2. **국내 경쟁 공백**: 팜모닝은 사용자가 많지만 사과 전문성이 얕고, 농사로는 UX가 나쁘며, KAMIS는 분석 기능이 없음. 이 세 가지의 장점을 결합하고 약점을 보완하는 포지션이 가능.

3. **해외 과수원 플랫폼의 한계**: Croptracker, Semios 등 과수원 특화 플랫폼도 "기존 과수원 관리"에 집중하며, "신규 과수원 설계/계획" 도구는 제공하지 않음.

4. **기상 기반 병해충 예측**: Semios(IoT)와 RIMpro(전문 모델)가 강력하지만, 한국 시장에는 진출하지 않았고 한국 병해충 모델도 없음. PJ18_APPLE이 한국 기상청 데이터로 한국 맞춤형 예측 모델을 구축할 기회.

5. **수익화 벤치마크**: 그린랩스의 어려움은 "범용 농업 플랫폼의 수익화"가 쉽지 않다는 점을 보여줌. PJ18_APPLE의 "사과 특화 + 유료 설계 도구" 전략이 더 명확한 가치 제안.

### 7.2 전략적 권고

#### 단기 (Phase 1-2)
- **밭 자동설계 + 수익 시뮬레이션을 MVP 핵심으로 유지** → 이것이 유일무이한 차별점
- **KAMIS API + 기상청 API 연동**을 통한 데이터 기반 서비스 구축
- **팜모닝과의 차별화**: 깊이(사과 전문) vs 넓이(범용)로 포지셔닝

#### 중기 (Phase 3)
- **기상 기반 병해충 예측** 고도화 (RIMpro/NEWA 모델 참고, 한국화)
- **3D 시각화**로 시각적 임팩트 극대화 (마케팅 효과)
- **커뮤니티** 구축 시 팜모닝의 사과 사용자를 타겟으로 이동 유도

#### 장기 (Phase 4+)
- **AI 병해충 이미지 진단** 추가 (Plantix 참고, 한국 사과 특화)
- **마켓플레이스**: 재배 데이터 기반 신뢰 보증 차별화
- **글로벌 진출 검토**: 일본/중국 사과 시장이 훨씬 크므로, 모델 확장 가능성
- **다과수 확장**: 배, 복숭아, 포도 등으로 "과수원 설계 플랫폼"으로 확장

### 7.3 최종 포지셔닝

```
PJ18_APPLE = "사과 농업의 네비게이션"

기존 서비스들이 "지도(정보)"만 제공한다면,
```
