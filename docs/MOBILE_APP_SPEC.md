## 1. 왜 앱이 필요한가

### 1.1 현장 환경 분석

| 상황 | 양손 상태 | 현재 대응 | 앱 해결 |
|------|-----------|-----------|---------|
| 적과 작업 (5월) | 가위+사과 | 작업 중단 후 폰 확인 | "오늘 날씨" 음성 명령 |
| 방제 작업 (6~9월) | 약통+호스 | 확인 불가 | "다음 방제 일정" 음성 안내 |
| 사다리 위 작업 | 사다리+과일 | 위험 (한 손 떼기) | 이어폰으로 음성 수신 |
| 수확 (9~11월) | 바구니+가위 | 쉬는 시간에만 확인 | "오늘 경매가" 실시간 음성 |
| 트랙터/SS기 운전 | 핸들 | 확인 불가 | "기상 경보" 자동 음성 알림 |
| 외국인 노동자 지시 | 시범+제스처 | 소통 어려움 | 음성 번역 (한→베트남/캄보디아) |

### 1.2 타겟 유저 디바이스

| 세그먼트 | 연령 | 디바이스 | 네트워크 | 특이사항 |
|----------|------|----------|----------|----------|
| 신규 농업인 | 30~40대 | 최신 스마트폰 | LTE/5G | 디지털 친숙 |
| 기존 농업인 | 50~60대 | 중저가 안드로이드 | LTE | 큰 글씨, 단순 UI 필요 |
| 고령 농업인 | 70대+ | 효도폰/구형 안드로이드 | 3G/LTE | 자녀 대리 설정, 음성 위주 |

**핵심**: 안드로이드 90%+ 점유 (농촌 지역), iOS는 후순위

### 1.3 웹 vs 앱 역할 분리

| 기능 | 웹 (현재) | 앱 (추후) |
|------|-----------|-----------|
| 밭 설계 + 시뮬레이션 | **주력** (큰 화면, 지도 조작) | 결과 조회만 |
| 급지 분석 | **주력** | 내 지역 급지만 |
| 기상 정보 | 상세 분석 | **주력** (현장 실시간 + 음성) |
| 경매가 | 차트 분석 | **주력** (실시간 알림 + 음성) |
| 병해충 경보 | 분석 | **주력** (푸시 알림 + 사진 진단) |
| 작업 캘린더 | 전체 보기 | **주력** (오늘 할 일 + 음성 안내) |
| 인력 관리 | 등록/관리 | 현장 연락 (탭→전화) |
| 비상연락망 | 목록 | **주력** (탭→즉시 전화) |

## 2. 음성 인터페이스 설계

### 2.1 핵심 원칙

```
1. 음성은 "보조"가 아니라 "주력" 입력 수단
2. 모든 핵심 기능은 음성만으로 완결 가능해야 함
3. 응답은 짧고 명확하게 (TTS로 들을 때 30초 이내)
4. 농업 용어 인식률 최우선 (적과, 전정, 방제, 탄저병 등)
5. 오프라인에서도 기본 음성 명령 동작
```

### 2.2 음성 명령 체계

#### 날씨/기상
| 명령 예시 | 응답 예시 |
|-----------|-----------|
| "오늘 날씨" | "청송 오늘 최저 2도, 최고 14도, 맑음. 서리 위험 없습니다." |
| "이번 주 비 오나" | "목요일 오후 강수확률 60%, 20mm 예상. 방제 일정 조정 권장." |
| "서리 위험 있어?" | "내일 새벽 영하 2도 예보. 방상팬 가동 준비하세요." |

#### 경매가/시장
| 명령 예시 | 응답 예시 |
|-----------|-----------|
| "오늘 사과값" | "안동 경매 후지 특등 8,200원. 어제보다 300원 올랐습니다." |
| "이번 주 시세" | "후지 특등 평균 7,900원. 지난주 대비 5% 상승." |
| "지금 팔아도 돼?" | "현재 시세 상승 추세. 이번 주 출하 유리합니다." |

#### 농작업
| 명령 예시 | 응답 예시 |
|-----------|-----------|
| "오늘 뭐 해야 돼" | "5월 3주차. 적과 2차 시기입니다. 오후 비 예보로 오전 작업 권장." |
| "다음 방제 언제" | "마지막 방제 5월 10일. 14일 경과. 이번 주 방제 필요합니다." |
| "탄저병 위험도" | "기온 27도, 습도 85%, 강수 있음. 탄저병 위험 높음. 방제 권장." |

#### 인력/연락
| 명령 예시 | 응답 예시 |
|-----------|-----------|
| "인력사무소 전화" | "영주 인력사무소 054-XXX-XXXX 연결합니다." → 자동 전화 |
| "농기센 전화" | "청송 농업기술센터 054-XXX-XXXX 연결합니다." → 자동 전화 |

#### 번역 (외국인 노동자용)
| 명령 예시 | 동작 |
|-----------|------|
| "베트남어로 '이 가지 잘라'" | TTS: "Cắt cành này" (베트남어 음성 출력) |
| "캄보디아어로 '물 줘'" | TTS: "ដាក់ទឹក" (크메르어 음성 출력) |

### 2.3 음성 기술 스택 (후보)

#### STT (음성→텍스트)

| 기술 | 한국어 성능 | 오프라인 | 비용 | 비고 |
|------|-------------|----------|------|------|
| **Whisper (OpenAI)** | 우수 | 로컬 모델 가능 | 무료(로컬)/저렴(API) | 농업 용어 파인튜닝 가능 |
| **Clova Speech (네이버)** | 최상 (한국어 특화) | 불가 | 유료 | 한국어 최강, 방언 지원 |
| **Google Cloud STT** | 우수 | 부분 지원 | 유료 | 다국어 지원 우수 |
| **On-device (Android)** | 보통 | 가능 | 무료 | 기본 명령만, 농업 용어 약함 |

**한국어 STT 벤치마크 (2026, CER 낮을수록 좋음)**:

| 서비스 | 평균 CER | 오프라인 | 비용 | 비고 |
|--------|----------|----------|------|------|
| **리턴제로 (Return Zero)** | **5.91%** | 불가 | 분당 과금 | 한국어 최고 정확도 |
| **Naver CLOVA Speech** | **7.52%** | 불가 | 분당 과금 (2025년 40% 인하) | 실시간 스트리밍 지원 |
| **ETRI** | 10.19% | 불가 | 무료(제한) | 정부 지원 |
| **OpenAI Whisper (API)** | ~11.45% | 불가 | $0.006/min | 다국어 |
| **Whisper (로컬, large-v3-turbo)** | ~11.45% | **가능** | 무료 | 농업 명령어 수준 충분 |
| **Picovoice Leopard** | 미측정 | **가능** | 유료 | 온디바이스 한국어 지원 |

**React Native STT 라이브러리**:

| 라이브러리 | 오프라인 | 한국어 | 비고 |
|-----------|----------|--------|------|
| `whisper.rn` | **완전 지원** | 지원 | whisper.cpp 바인딩, 무료 |
| `@jamsch/expo-speech-recognition` | OS 의존 | 지원 | Expo 최적 통합 |
| `react-native-voice` | 부분 | 지원 | 성숙한 라이브러리 |
| React Native ExecuTorch | **완전 지원** | 지원 | 신규 (v0.3.0), 유망 |
| Picovoice Leopard | **완전 지원** | 지원 | 상용, 고품질 |

**권장**: CLOVA Speech (온라인 고정밀 7.52%) + Whisper 로컬 via `whisper.rn` (오프라인 폴백 11.45%)

#### TTS (텍스트→음성)

| 기술 | 자연스러움 | 한국어 | 비용 | 비고 |
|------|------------|--------|------|------|
| **PJ01 TTS 시스템** | 우수 | 커스텀 | 기구축 | Mirai 음성, 캐릭터 일관성 |
| **Clova Voice (네이버)** | 최상 | 네이티브 | 유료 | 다양한 화자, 감정 조절 |
| **Google Cloud TTS** | 우수 | 좋음 | 유료 | 다국어 지원 |
| **Edge TTS (MS)** | 좋음 | 좋음 | 무료 | 오프라인 불가 |
| **On-device (Android)** | 보통 | 기본 | 무료 | 오프라인 가능 |

**권장**: PJ01 TTS (Mirai 브랜드) + Android 기본 TTS (오프라인 폴백)

### 2.4 음성 UX 흐름

```
[대기]
  ↓ "헤이 사과" 또는 버튼 탭
[듣기] 🎤 마이크 활성 (파형 애니메이션)
  ↓ 사용자 발화
[처리] ⏳ STT → 의도 분석 → 데이터 조회
  ↓ 0.5~2초
[응답] 🔊 TTS 음성 출력 + 화면 카드 표시
  ↓
[대기] 5초 후 자동 닫힘 또는 "다음 질문" 연속 대화
```

**웨이크워드 후보**: "헤이 사과", "미라이", "농장아"

## 4. 기술 아키텍처

### 4.1 앱 프레임워크 선택

| 옵션 | 장점 | 단점 | 적합도 |
|------|------|------|--------|
| **React Native** | 웹 코드 재사용, 네이티브 성능, 큰 생태계 | 네이티브 브릿지 필요 | ★★★★★ |
| **Flutter** | 성능 최상, 일관된 UI | 웹 코드 재사용 불가, Dart 학습 | ★★★ |
| **PWA** | 웹 그대로 배포, 설치 없음 | 음성/카메라/푸시 제한, 오프라인 약함 | ★★ |
| **Kotlin/Swift 네이티브** | 최고 성능, 모든 기능 | 개발 비용 2배, 유지보수 부담 | ★★ |

**권장 전략 (단계적)**:
- **즉시 (웹 단계)**: Next.js PWA로 Serwist 오프라인 캐싱 + Web Speech API 기본 음성
  - 참고: Next.js 16 Turbopack은 Serwist 비호환 → PWA 빌드 시 Webpack 사용 필요
- **앱 전환 시**: React Native + Expo
  - Next.js → RN 컴포넌트/로직 최대 재사용
  - `whisper.rn` 오프라인 한국어 STT
  - 네이티브 카메라 (병해충 AI 진단)
  - FCM 푸시 알림 완전 지원

### 4.3 오프라인 전략

농촌 지역 특성상 네트워크 불안정 빈번 → 오프라인 우선 설계 필수

| 기능 | 온라인 | 오프라인 | 캐시 주기 |
|------|--------|----------|-----------|
| 오늘 날씨 | 실시간 | 마지막 캐시 (3시간) | 3h |
| 경매가 | 실시간 | 마지막 캐시 (6시간) | 6h |
| 작업 캘린더 | 서버 동기화 | 로컬 SQLite | 일 1회 |
| 비상연락망 | 서버 동기화 | 로컬 저장 (항상 사용 가능) | 주 1회 |
| 음성 인식 | Clova/Whisper API | Whisper 로컬 모델 | 상시 |
| 음성 출력 | PJ01 TTS | Android 기본 TTS | 상시 |
| 병해충 진단 | AI API | 불가 (사진 저장 후 동기화) | — |

### 4.4 푸시 알림 체계

| 알림 유형 | 트리거 | 긴급도 | 음성 안내 |
|-----------|--------|--------|-----------|
| 서리 경보 | 최저기온 < 0°C (개화기) | 🔴 긴급 | 자동 TTS |
| 폭우 경보 | 일강수 > 50mm | 🔴 긴급 | 자동 TTS |
| 병해충 고위험 | 기온+습도+강수 임계 | 🟡 주의 | 탭 시 TTS |
| 경매가 급변 | 전일 대비 ±10% | 🟡 주의 | 탭 시 TTS |
| 오늘 작업 안내 | 매일 오전 6시 | 🟢 정보 | 탭 시 TTS |
| 방제 시기 알림 | 마지막 방제 후 14일 | 🟢 정보 | 탭 시 TTS |

## 5. 번역 기능 (외국인 노동자 지원)

### 5.1 배경

- 사과 농가 인력의 상당수가 외국인 노동자 (베트남, 캄보디아, 태국 등)
- 의사소통 어려움 → 작업 효율 저하, 안전사고 위험
- 현장에서 바로 음성 번역이 가능하면 실질적 가치

### 5.2 지원 언어 (우선순위)

| 순위 | 언어 | 대상 | 농업 노동자 비율 |
|------|------|------|-----------------|
| 1 | 베트남어 | 베트남 | 가장 많음 |
| 2 | 크메르어 (캄보디아) | 캄보디아 | 증가 추세 |
| 3 | 태국어 | 태국 | 일부 |
| 4 | 미얀마어 | 미얀마 | 일부 |

### 5.3 번역 시나리오

**사전 등록 농작업 문구** (빠른 번역):
```
적과:     "이 가지에서 작은 사과 따세요"
전정:     "이 가지 잘라주세요"
수확:     "빨간 것만 따세요"
안전:     "사다리 조심하세요"
관수:     "물 틀어주세요 / 꺼주세요"
```

**실시간 번역** (음성→음성):
```
한국어 음성 → STT → 번역 API → TTS → 외국어 음성
```

### 5.4 기술

| 기술 | 용도 | 비고 |
|------|------|------|
| Papago API (네이버) | 한↔베트남/크메르/태국 | 한국어 기반 번역 최상 |
| Google Translate API | 다국어 폴백 | 크메르어 지원 |
| 사전 등록 문구 DB | 오프라인 즉시 번역 | 농작업 특화 100+ 문구 |

## 6. AI 병해충 진단 (카메라)

### 6.1 사용 흐름

```
[카메라 버튼] → 사진 촬영 → AI 분석 (1~3초) → 결과 + 방제 가이드
```

### 6.2 진단 대상

| 병해충 | 시각적 특징 | 난이도 |
|--------|-------------|--------|
| 탄저병 | 과실 갈색 함몰 반점 | 쉬움 |
| 갈색무늬병 | 잎에 갈색 둥근 반점 | 중간 |
| 겹무늬썩음병 | 과실 동심원 무늬 부패 | 쉬움 |
| 점무늬낙엽병 | 잎에 작은 갈색 점 | 중간 |
| 진딧물 | 신초 뒤틀림, 벌레 군집 | 쉬움 |
| 응애 | 잎 탈색, 미세 벌레 | 어려움 |

### 6.3 기술

| 접근법 | 설명 | 정확도 | 비용 |
|--------|------|--------|------|
| **GPT-4o Vision** | 사진→설명 요청 | 높음 | API 비용 |
| **커스텀 CNN** | 직접 학습 (데이터 필요) | 최상 (학습 후) | 학습 비용 + 데이터 수집 |
| **Plantix API** | 기존 농업 AI | 중간 | 유료 |

**권장**: 초기 GPT-4o Vision → 데이터 축적 후 커스텀 모델

- 구글 플레이 스토어 등록

## 9. 경쟁 환경 & 차별점 (2026 리서치)

### 국내 경쟁

| 앱 | 사용자 | 음성 | 오프라인 | PJ18 차별점 |
|----|--------|------|----------|-------------|

| **농사로** | 정보 포털 | 없음 | 없음 | 현장 실시간 + UX |

팜모닝: 투자 2,485억원, AI 챗봇 추가했으나 음성 인터페이스는 없음 → **음성이 가장 큰 차별점**

### 글로벌 트렌드

| 앱 | 국가 | 특징 |
|----|------|------|
| **Bharat-VISTAAR** | 인도 (2026.02) | 정부 주도 농업 AI, 음성 기반, 다국어 |
| **Plantix** | 글로벌 (베를린) | 400+ 병해 진단, 18개 방언, 음성 명령 |

**글로벌 흐름**: 농업 = 음성 퍼스트의 최적 도메인 (더러운/젖은 손, 야외 강한 햇빛, 고령자 시력 저하)

### 핵심 UX 주의사항 (학술 리서치 기반)

```
⚠️ "음성 전용"이 아닌 "음성 보조" 설계 필수
- 사투리/방언 인식 어려움 (경북 사투리 등)
- 농기계 소음 환경에서 인식률 저하
- 발화 장애가 있는 고령자 존재
→ 음성 = 주요 단축키, 터치/시각 = 항상 대안 제공
→ 멀티모달 (음성+터치+시각) 접근이 최선
```
