# PJ18_APPLE - 사과 농업인 전문 플랫폼

> **프로젝트 코드명**: PJ18_APPLE
> **최초 작성일**: 2026-02-16
> **상태**: 기획 단계
> **한 줄 요약**: 사과 재배 전 과정을 데이터 기반으로 지원하는 농업인 전문 플랫폼

---

## 1. 프로젝트 비전

### 1.1 문제 정의

| 문제 | 현재 상황 | 영향 |
|------|-----------|------|
| 정보 분산 | 품종 정보, 재배법, 가격 정보가 각각 다른 곳에 흩어져 있음 | 농업인이 의사결정에 필요한 정보를 모으는 데 과도한 시간 소요 |
| 경험 의존 | 재배 기술이 구전/경험 기반으로 전수 | 신규 농업인 진입 장벽 높음, 세대 단절 위험 |
| 가격 불투명 | 경매장 가격 정보가 실시간 접근 어려움 | 출하 시기/물량 의사결정 부정확 |
| 설계 부재 | 밭 설계를 감으로 진행, 전문 설계 서비스 없음 | 비효율적 토지 활용, 동선 낭비, 수익성 저하 |
| 리스크 대응 | 기후 변화, 병해충, 국제 시장 변동에 대한 체계적 대응 없음 | 피해 발생 후 사후 대응만 가능 |

### 1.2 해법 요약

**데이터 기반 의사결정 플랫폼**: 기상·경매·통계 데이터를 통합하고, AI 기반 밭 설계와 수익 시뮬레이션을 제공하여 사과 농업의 전 과정(계획→재배→수확→판매)을 과학적으로 지원

### 1.3 타겟 유저

| 세그먼트 | 특징 | 핵심 니즈 |
|----------|------|-----------|
| **신규 농업인 (30~40대)** | 귀농/전업, 디지털 친숙 | 밭 설계, 수익 시뮬레이션, 체계적 재배법 |
| **기존 농업인 (50~60대)** | 경험 풍부, 스마트폰 사용 가능 | 경매가 실시간 확인, 병해충 알림, 기상 예보 |
| **고령 농업인 (70대~)** | 기술 접근성 낮음 | 극도로 단순한 알림 (자녀/조합 통해 간접 사용) |
| **농업 법인/조합** | 대규모 운영 | 통합 데이터, 다필지 관리, 리포트 |

- 기상청 API 연동으로 동적 일정 조정
- 푸시 알림: "이번 주 OO지역 적과 적기입니다"

**주요 병해충**:

| 병해충 | 발생 조건 | 방제 시기 | 피해 규모 |
|--------|-----------|-----------|-----------|
| **탄저병** | 고온다습 (7~8월), 25℃ 이상 + 강우 | 6월 하순~8월, 2주 간격 | 심하면 수확량 50%+ 감소 |
| **갈색무늬병 (갈반병)** | 다습 환경, 낙엽 전염원 | 5월~9월 | 조기 낙엽 → 이듬해 수확 영향 |
| **겹무늬썩음병** | 장마철 이후 고온 다습 | 6~9월 | 저장 중 부패 확대 |
| **부란병** | 동해/상처 부위 감염 | 연중 (예방 위주) | 나무 고사 위험 |
| **점무늬낙엽병** | 다습 환경, 비 이후 | 5~8월 | 조기 낙엽, 수세 약화 |
| **진딧물** | 4~6월 신초 생장기 | 4월 하순~6월 | 신초 위축, 그을음병 유발 |
| **응애** | 고온 건조 (7~8월) | 6~8월 | 잎 피해, 광합성 저하 |
| **복숭아심식나방** | 6~9월 | 6월~9월, 성페로몬 트랩 연동 | 과실 내부 식해 |

**기상 API 연동 로직 (핵심)**:
```
IF 기온 >= 25℃ AND 강수량 > 30mm/주 AND 습도 > 80%
  → 탄저병 위험도: HIGH
  → 알림: "탄저병 방제 시기입니다. 추천 약제: 프로피네브 수화제"
  → 마지막 방제 후 14일 경과 시 재방제 권고

IF 최저기온 < 0℃ AND 개화기 (4월)
  → 서리 피해 경고: CRITICAL
  → 알림: "내일 새벽 서리 예보. 방상팬/살수법 가동 준비"
```

#### 2.1.4 지역별 적합 품종 가이드

**지역 분류 및 품종 적합도**:

| 지역 | 기후 특성 | 추천 품종 | 비고 |
|------|-----------|-----------|------|
| **경북 (영주, 안동, 청송)** | 일교차 큼, 해발 높음 | 부사, 감홍, 시나노골드 | 전통 사과 명산지, 착색 우수 |
| **충북 (충주, 제천)** | 내륙성 기후, 일교차 | 부사, 홍로, 아리수 | 중부 사과 산지 확대 중 |
| **경남 (거창, 함양)** | 산간 지역 일교차 | 부사, 홍로 | 고랭지 사과 |
| **강원 (양구, 정선)** | 고랭지, 냉량 | 부사, 감홍 | 기후 변화로 재배 북상 중 |
| **전북 (장수, 무주)** | 고랭지, 일교차 | 부사, 홍로 | 신흥 사과 산지 |

**기후 변화 트렌드**:
- 사과 재배 적지가 연간 약 1~2km씩 북상 중
- 2050년 전망: 강원 고랭지가 주산지 될 가능성
- 고온 적응 품종 (아리수 등) 수요 증가 예상

### 2.3 밭 자동 설계 시스템 (Farm Auto-Designer) ⭐ 킬러 피처

#### 2.3.1 설계 프로세스

```
[Step 1] 토지 선택
  └→ 지도(Leaflet/Mapbox)에서 필지 선택 또는 주소 검색
  └→ 토지 정보 자동 로드 (면적, 지가, 경사도, 방위)
  └→ 토지 경계 내에서 밭으로 사용할 영역 드래그 지정

[Step 2] 재배 스타일 선택
  ├→ [A] 밀식재배 (고밀도): 열간 3.5m × 주간 1.5~2m
  │    - 나무 높이 목표: 3~3.5m (방추형)
  │    - M.9 왜성대목 사용
  │    - 장점: 조기 수확 (3년차~), 단위면적 수확량 높음
  │    - 단점: 초기 투자 높음, 기술 요구도 높음
  │
  ├→ [B] 반밀식재배 (중밀도): 열간 4~5m × 주간 2~3m
  │    - 나무 높이 목표: 3.5~4m
  │    - M.26 반왜성대목
  │    - 장점: 관리 용이, 균형잡힌 투자/수익
  │    - 단점: 밀식 대비 수확량 낮음
  │
  └→ [C] 일반재배 (저밀도): 열간 5~6m × 주간 4~5m
       - 나무 높이 목표: 4~5m (개심자연형)
       - 실생대목 또는 MM.106
       - 장점: 초기 투자 낮음, 전통 방식
       - 단점: 수확 시작 5~6년, 관리 면적 넓음

[Step 3] 품종 선택
  └→ 품종별 수분수 배치 자동 계산
  └→ 혼식 비율 추천 (주품종 80% + 수분수 20%)

[Step 4] 자동 설계 생성
  ├→ 나무 배치도 (Grid Layout 알고리즘)
  ├→ 동선 설계 (입구 위치 → 농기계 통행로 → 회전 반경)
  ├→ 관수 시스템 (수원 위치 → 주관 → 지관 → 스프링클러/점적관수)
  ├→ 방풍림 위치 추천
  └→ 방상팬 최적 위치

[Step 5] 수익 시뮬레이션
  └→ (별도 섹션 2.3.3 참조)

[Step 6] 대안 설계
  └→ "다른 대안 보기" → 스타일/밀도/품종 조합 변경하여 재설계
  └→ A안 vs B안 비교표 자동 생성

[Step 7] 3D 시각화
  └→ Three.js 기반 간단한 3D 렌더링
  └→ 계절별 모습 시뮬레이션 (봄 개화 → 여름 녹음 → 가을 수확)
```

#### 2.3.2 자동 설계 알고리즘

**핵심 알고리즘 (주차장 자동설계와 유사)**:

| 알고리즘 | 용도 | 비고 |
|----------|------|------|
| **Bin Packing** | 주어진 면적에 나무 최대 배치 | 열간·주간 간격 제약 조건 |
| **Grid Layout + Constraint Solver** | 경사도/방위 고려한 열 방향 결정 | 남북 방향 열 우선 (일조량) |
| **A* Pathfinding** | 농기계 동선 최적화 | SS기(스피드스프레이어) 회전반경 3~4m 고려 |
| **Flow Network** | 관수 시스템 배관 최적 경로 | 수원→주관→지관, 수압 계산 |
| **Voronoi Diagram** | 스프링클러 커버리지 최적화 | 관수 반경 기반 배치 |

**참고 오픈소스**:
- 주차장 자동설계: Parking Lot Layout Optimization (학술 자료 다수)
- 농업용 GIS: QGIS 플러그인 생태계
- 3D 렌더링: Three.js (웹 기반, 가벼움)
- 지도: Leaflet + OpenStreetMap (무료) 또는 Mapbox
- 지리 연산: Turf.js (면적, 거리, 포함관계 등)

#### 2.3.3 수익 시뮬레이션 엔진

**비용 모델**:
```
총 비용 = 토지비 + 묘목비 + 자재비 + 농약비 + 비료비 + 인건비 + 기계비 + 관수시설비

항목별 추정:
- 토지비: 공시지가 API 연동 (국토부)
- 묘목비: 품종별 × 본수 (밀식 M.9 묘목 약 8,000~15,000원/주)
- 자재비: 지주, 철선, 반사필름 등
- 농약비: 연간 약 15~20회 방제, 회당 비용 × 면적
- 비료비: 기비 + 추비, 면적당 계산
- 인건비: 지역별 농업 일당 × 작업별 필요 인일수
- 기계비: SS기, 관리기 등 (임차 vs 보유)
- 관수시설: 점적관수 vs 미니스프링클러 비용 차이
```

**수익 모델 (품종별 × 연차별)**:
```
0~2년차: 투자기 (수익 0, 비용만 발생)
3년차:   첫 수확 (밀식 기준), 정상 수확량의 20~30%
4~5년차: 수확량 증가, 정상의 50~70%
6~10년차: 최성기, 정상 수확량 100%
11~15년차: 유지기, 수확량 90~100%
16~20년차: 쇠퇴 시작, 수확량 감소 + 과실 품질 저하 → 가격 하락
20년차~: 개식(나무 교체) 검토 시기
```

**시각화 산출물**:
- 연차별 투자/수익 그래프 (막대+라인 복합 차트)
- 누적 손익분기점 표시
- 품종 A vs B 비교 차트
- 시나리오 분석: 낙관/중립/비관 (경매가 ±20% 변동)

### 3.2 경쟁 분석

| 서비스 | 특징 | 장점 | 약점 | 차별화 포인트 |
|--------|------|------|------|--------------|
| **팜모닝** | 범용 농업 커뮤니티 앱 | 사용자 많음, 커뮤니티 활발 | 사과 전문성 없음, 깊이 부족 | 사과 특화 깊이 |
| **농사로** | 농촌진흥청 공식 | 신뢰도 높음, 데이터 풍부 | UI 불편, 업데이트 느림, 실시간성 없음 | 실시간 + UX |
| **KAMIS** | 가격 정보 | 공식 가격 데이터 | 가격 정보만 제공, 분석 없음 | 가격 + 분석 + 예측 |
| **엑셀/수첩** | 개인 관리 | 익숙함 | 확장 불가, 공유 불가, 분석 한계 | 자동화 + 시각화 |

### 3.3 핵심 차별화 요소 (경쟁 우위)

1. **밭 자동설계** → 국내 유일 (어디에도 없음)
2. **수익 시뮬레이션** (품종별 × 연차별) → 투자 의사결정 도구
3. **기상 연동 병해충 예측** → 실질적 피해 방지 가치
4. **경매장 가격 + 분석** → 출하 의사결정 지원
5. **사과 특화** → 범용 서비스 대비 깊이 있는 전문성
